# AGENTS.md — Agent Coordination and Strategy

This document outlines the strategy for AI agents working within the `mkagent` monorepo.

## Project Intelligence
- **Architecture**: Monorepo managed by Turbo.
- **Intelligence Layer**: Centralized Project Intelligence in `packages/mkagent/src/detect.ts`.
- **Documentation**: Professional, technical, and LLM-parsable.

## Tech Stack and Architecture
### CLI Architecture
- **Command Layer**: `commander` based CLI Entrypoint (`index.ts`).
- **Interaction Layer**: `@clack/prompts` for a premium terminal UX.
- **Intelligence Layer**: Deep scan logic returning `ProjectIntelligence`.
- **Execution Layer**: AI-driven file generation and Next.js/Vite scaffolding.

### Docs Architecture
- **Framework**: React 18 with Vite.
- **Styling**: Tailwind CSS with a custom "Dev Docs" theme.
- **Navigation**: React Router with nested routes.

## Core Workflows (Commands)
- **Scaffold New Project**: `mkagent init`
- **Check Environment Health**: `mkagent doctor`
- **Update Existing Agent Files**: `mkagent regenerate`

## Structural Mental Model
- Agents should think in terms of "Project Intelligence". 
- Every file generated by `mkagent` should answer: "What does an AI agent need to know to work here effectively?"
- The goal is to eliminate the "Context Gap" for developer agents.

## Guardrails and Safety
- Ensure `detect.ts` is robust against various project structures (monorepos, nested apps).
- Validate API availability before starting generation.
- Never log raw API keys.

## Agent Personality and Instructions
- Provide deep technical insights into project-specific logic (e.g., App Router handling in Next.js).
- Avoid generic "This is a React app" descriptions.
- Use clean, professional English optimized for parsing.
